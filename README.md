# 1. 프로젝트 선정 이유
#### 
대학 전공 선택은 개인의 진로와 직결되는 매우 중요한 결정입니다.
그러나 많은 학생들이 자신의 성향을 충분히 분석하지 못한 채 성적이나 주변의 권유에 따라 전공을 선택하는 경우가 많습니다.

본 프로젝트는 이러한 문제점을 개선하고자,
사용자의 성향을 수치화하여 데이터 기반으로 분석하고 이에 적합한 대학 전공을 추천하는 프로그램을 구현하는 것을 목표로 선정되었습니다.

특히 본 프로젝트는 다음과 같은 특징을 가집니다.

- 객관적인 수치 기반 전공 추천
- 사용자 만족도 피드백을 반영한 반복 추천 구조
- 주전공 확정 이후 복수전공까지 고려한 확장 설계

이를 통해 실제 전공 탐색 과정과 유사한 추천 시스템을 구현하고자 하였습니다.

# 2. 코드 설명

### 1. 전공 데이터 구성
본 프로그램에서는 총 30개의 대학 전공 데이터를 사전에 정의하였습니다.
각 전공은 다음 9가지 기준을 바탕으로 1~10 스케일의 점수로 수치화되어 있습니다.

- 팀플 선호
- 프로젝트 선호
- 학습량
- 취업 선호
- 발표 선호
- 창의성
- 실습 선호
- 진로 다양성
- 사고력

이를 통해 각 전공의 특성을 정량적으로 표현하였으며,
해당 데이터는 이후 KNN 기반 전공 추천 알고리즘의 입력값으로 활용됩니다.

### 2. 사용자 성향 입력
사용자는 자신의 학습 및 활동 성향을 각 기준별로 1~10 스케일로 입력하게 됩니다.
입력값이 범위를 벗어날 경우 재입력을 요구하도록 하여,
잘못된 입력으로 인한 프로그램 오류를 방지하였습니다.
입력된 사용자 성향 데이터는 이후 모든 추천 과정의 기준 벡터로 사용됩니다.


### 3. KNN 모델 초기화 및 학습
전공 추천을 위해 KNN(K-Nearest Neighbors) 알고리즘을 사용하였습니다.
StandardScaler를 활용하여 전공 데이터를 정규화하였습니다.
유클리디안 거리(Euclidean Distance)를 기준으로 사용자 성향과 전공 간의 유사도를 계산합니다.
전체 전공 데이터를 기반으로 KNN 모델을 학습시켰습니다.
이를 통해 사용자 성향과 가장 유사한 전공을 효과적으로 탐색할 수 있도록 구현하였습니다.


### 4. 전공 추천 함수
사용자의 성향 벡터를 학습된 KNN 모델에 적용하여
적합도가 높은 전공 TOP 5를 도출합니다.
- 이미 추천된 전공은 제외 목록(set)으로 관리됩니다.
- 거리 값을 기반으로 적합도를 계산하여 백분율 형태로 변환합니다.
- 학과명, 전공 분류, 적합도를 표 형태로 출력합니다.


### 5. 만족도 입력 및 피드백 수집
추천된 전공에 대해 사용자는 만족도(1~10)를 입력하게 됩니다.
- 만족도가 10점인 전공이 존재할 경우 해당 전공을 주전공으로 확정합니다.
- 만족도 10점인 전공이 존재하지 않을 경우 다음 단계로 진행합니다.


### 6. 성향 업데이트 함수
최초 추천 과정에서 주전공이 확정되지 않을 수 있으므로,
입력된 만족도를 기존 사용자 성향에 반영하여 성향을 조정합니다.

조정된 성향을 기반으로 다시 전공 추천을 수행함으로써,
사용자의 취향을 점진적으로 반영하는 추천 구조를 구현하였습니다.

### 7. 메인 함수에 대한 설명
메인 함수에서는 프로그램의 전체 흐름을 제어합니다.
1. 사용자 성향 입력
2. 제외할 전공 목록(set) 초기화
3. 주전공이 확정될 때까지 while문을 통한 반복 추천
4. 추천 결과를 표 형태로 출력
5. 만족도 입력 및 성향 업데이트 수행
6. 주전공 확정 시 복수전공 추천 여부 확인 후 프로그램 종료

# 프로젝트 결론 및 총평
#### 
앞서 언급드린 바와 같이, 이 프로젝트에서 쓰인 프로그램은 게임의 갯수가 많아질수록 더 확실한 이득을 보는 구조를 가지고 있습니다. 게임의 갯수가 적으면 모든 게임을 다 해보면서 자기 성향과 맞는 게임을 하면 되지만, 게임의 갯수가 늘어날수록 물리적인 한계가 있기 때문입니다. 또한, 다인원이 게임을 진행하고 싶을 때 어떤 게임을 할 지 애매한 경우들이 존재합니다. 이 때, 서로 토의하여 게임에서 요구하는 능력에 대한 선호도를 정하여 입력한다면 게임을 정하는 과정이 굉장히 수월해집니다. 게다가, 최초에 추천받은 게임에 흥미를 느끼지 못했다고 하더라도, 후순위로 추천해주는 게임들의 목록이 작성되기 때문에 이 과정에 있어서 시간을 굉장히 줄일 수 있습니다. 하지만, 문제점 또한 명확하다고 생각합니다. 가장 먼저 예상되는 문제점으로는 게임의 종류가 다양해질수록 선제적으로 입력해야하는 데이터가 많아지고 또 각 게임의 특징들을 스케일에 따라서 조율해야 하기 때문에 이 부분에서 주관이 개입될 수 있습니다. 두 번째로는, 코드의 소요 시간과 관련된 문제입니다. KNN 모델을 사용하기 때문에 데이터 전처리 과정에서 소요되는 시간보다 데이터 후처리 과정에서 소요되는 시간이 더 크다는 점은 당연합니다. 그러나, 게임의 종류가 많아지고 이에 대한 정보들이 선제적으로 다수 입력될수록 데이터 후처리 과정에서 소요되는 시간이 점점 오래걸리게 될 것입니다. 만약, 정말 오랜시간이 소요된다면, 머신러닝을 사용하는 과정에 있어서 KNN 모델이 아니라, 다른 모델로 이 프로그램을 구현하는 것이 더 유리하지 않나라는 의문점을 품을 수 있다고 생각합니다. 그러나, 개인적인 의견을 첨언하자면, 데이터 후처리과정에서 소요되는 시간이 아무리 오래걸린다고 하더라도 모든 게임을 직접 다 해본 뒤 알맞는 게임을 찾는 시간보다는 적게 걸릴 것입니다. 그렇기에, 이에 대한 문제점은 크게 신경쓰이지 않는다는 것이 저의 사견입니다. 결론적으로, 이러한 추천 알고리즘을 만드는 과정에서는  ‘Train set, Test set을 이용하여 Regression 문제를 푸는 방법’ 혹은 ‘CNN 모듈을 사용하는 방법’ 등 다른 머신러닝적 요소들이 적용된 method들은 KNN 모델보다 크게 유리하지 않다고 생각합니다. 이러한 제 판단이 이번 프로젝트를 위와 같은 과정을 통해 설계하게 만들었고 기회가 된다면 더 유용한 모델에 대해서 배워보고 싶다는 생각을 가지게 된 계기가 되었습니다.
